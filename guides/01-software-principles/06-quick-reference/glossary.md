# Glossary

> **One-line summary**: Definitions for recurring terminology used throughout the guide.

## Terms
- **Action Class**: Invokable class encapsulating a single use case.
- **Decorator**: Wrapper that extends behavior while conforming to the same interface.
- **DIP**: Dependency Inversion Principle—depend on abstractions, not concretions.
- **DRY**: Don’t Repeat Yourself—single source of truth.
- **Eager Loading**: Loading related models upfront to avoid N+1 queries.
- **Feature Flag**: Toggle controlling access to functionality at runtime.
- **ISP**: Interface Segregation Principle—prefer narrow contracts.
- **LSP**: Liskov Substitution Principle—subtypes must be substitutable.
- **OCP**: Open/Closed Principle—open for extension, closed for modification.
- **Observer**: Pattern allowing event subscribers to react to state changes.
- **Pipeline**: Sequence of operations (middleware-like) where each step processes input then forwards it.
- **Repository**: Abstraction over data persistence operations.
- **Service Layer**: Domain orchestration tier between controllers and infrastructure.
- **SRP**: Single Responsibility Principle—one reason to change.
- **Strategy**: Pattern encapsulating interchangeable algorithms.
- **YAGNI**: You Ain’t Gonna Need It—avoid speculative implementation.
